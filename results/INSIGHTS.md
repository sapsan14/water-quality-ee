## Мини-глоссарий

- **Выборка / проба** — одна запись о замере воды в точке и времени.
- **Признак (feature)** — число или категория, которое подаём модели на вход (лабораторные показатели, дата, домен и т.д.).
- **Целевая переменная (target)** — то, что предсказываем: здесь `compliant` (1 = соответствует норме, 0 = нарушение).
- **EDA (Exploratory Data Analysis)** — разведочный анализ: смотрим таблицы, графики, пропуски, баланс классов, без обучения модели.
- **Корреляция** — насколько два числовых ряда тянутся «в одну сторону» (линейно). Не доказывает причинность.
- **Обучение (train) и тест (test)** — данные делятся на две части: на одной модель **учится**, на другой мы **честно проверяем** качество. Если учиться на всём сразу — оценка завышается (подгонка под «экзамен»).
- **Стратификация** — при разбиении train/test сохраняем **долю классов** (чтобы и там, и там было мало нарушений в одинаковой пропорции).
- **Импутация** — заполнение пропусков (например, медианой по обучающей выборке).
- **Масштабирование (StandardScaler)** — приводит признаки к сопоставимому масштабу (среднее 0, разброс 1); важно для многих моделей, особенно логистической регрессии.
- **Утечка данных (leakage)** — если для заполнения пропусков или масштаба использовали **весь** датасет включая test, информация из «будущего экзамена» попала в обучение.
- **Precision / Recall (для класса «нарушение»)**  
  - *Precision* — из того, что модель назвала нарушением, какая доля действительно нарушения.  
  - *Recall* — из всех **реальных** нарушений, какую долю модель **не пропустила**. Для безопасности воды критичен **Recall по классу 0**.
- **ROC-AUC** — насколько хорошо модель ранжирует «нарушение выше нормы» без привязки к одному порогу; 1.0 — идеально, 0.5 — случайный уровень.
- **Confusion matrix** — таблица «факт vs предсказание» (TP, TN, FP, FN). **False Negative** по нарушению — сказали «норма», а вода опасна.
- **Feature importance (Random Forest)** — насколько в среднем деревья использовали признак для уменьшения ошибки; ориентир, не «физическая причина».
- **Уезд (`county`) и `county_source`** — после `load_all()` колонка `county` может быть заполнена не только из XML: см. `src/county_infer.py` и справочник `data/reference/location_county_overrides.csv`. Столбец `county_source` показывает, откуда взялось значение (`xml`, `override`, `geocode`, `unknown`). В матрицу признаков для модели попадает закодированный уезд (`county_encoded`), а не `county_source`.

Это **сквозной учебный текст**: от сырых XML до двух моделей, с объяснением терминов в markdown между ячейками.

- После полного прогона вы увидите те же этапы, что в 01–05, но в одном файле: загрузка → осмотр → признаки → train/test → логистическая регрессия и Random Forest → сравнение.

### Что делают основные кодовые блоки (по смыслу)

| Блок | Простыми словами |
|------|------------------|
| Импорты | Подключаем pandas, визуализацию и модули проекта (`data_loader`, `features`, sklearn). |
| `load_domain` | Скачивает/читает кэш XML и превращает его в таблицу `DataFrame`. |

| `RandomForestClassifier` | Много деревьев решений; улавливает нелинейности, обычно точнее. |
| `evaluate_model` / сравнение | Метрики, отчёт classification_report, ROC-AUC. |
| Feature importance | Какие столбцы RF использовал чаще всего при разбиениях. |

### Пояснение коду по разделам

4. **`info`** — типы и число непустых значений по каждому столбцу.
5. **`describe`** — минимум, максимум, квартили, среднее для числовых колонок (чувствительно к выбросам).
6. **Доля пропусков** — `isnull` + проценты: где данных не хватает для модели.
7. **`plot_class_distribution`** — столбиковая диаграмма: сколько «0» и «1»; показывает дисбаланс.
8. **`add_time_features`** — новые столбцы из даты; дальше группировки/графики по времени и сезону.
9. **География** — если `county` есть, считаем нарушения по уездам; иначе сообщение в духе «данных нет».
10. **Гистограммы / KDE** — как распределены измерения у «норма» vs «нарушение».

4. **`plot_confusion_matrix`** — тепловая карта 2×2 для каждой модели.

## Ноутбук 06 — продвинутые модели (`06_advanced_models.ipynb`)

### Зачем этот ноутбук

Эксперименты **поверх** базовой цепочки 03–05: темпоральное разбиение по годам, **LightGBM** с пропусками без `StandardScaler`, **калибровка** вероятностей (`CalibratedClassifierCV`), **SHAP**, **TimeSeriesSplit (5 фолдов)** (`evaluate.temporal_cv_metrics`) и **порог** под precision по нарушениям (`best_threshold_max_recall_at_precision`). Данные: `load_all()` (в т.ч. **basseinid**) + `build_dataset()` + `county_infer`.

### Результаты и смысл (кратко)

- Темпоральный тест (поздние годы) жёстче случайного split — метрики обычно **ниже**, чем в таблице из `04_models.ipynb`; это ожидаемо, а не «поломка» модели.
- LightGBM + калибровка дают рабочий компромисс под задачу «не пропустить нарушение»; точные цифры смотрите в выводе ячеек ноутбука и в **`docs/report.md`** (раздел 6b).
- SHAP дополняет «важность по деревьям»: видно вклад признаков с учётом контекста.

### Что делают основные блоки (по смыслу)

| Блок | Простыми словами |
|------|------------------|
| `load_all` + `build_dataset` | Та же матрица признаков, что в 03–05; уезд уже может быть частично заполнен. |
| Отбор train/test по `year` | Учимся на ранних годах, проверяем на поздних — имитация «прогноза в будущее». |
| LightGBM | Градиентный бустинг с нативными пропусками; без масштабирования, как в ячейках ноутбука. |
| `CalibratedClassifierCV` | Подгонка `predict_proba` под реальные доли классов (изотоника или сигмоида). |
| SHAP | Объяснение вклада признаков для отдельных предсказаний и в среднем. |

## Что вынести в итог курсовой / отчёта

1. **Данные** реальные, но **XML opendata неполон** по ряду полей (география в XML, часть химии); уезд **частично** восстанавливается справочником/геокодером — это нужно проговорить как ограничение и как улучшение по сравнению с «только XML».
2. **Класс нарушений редкий** — без `class_weight` или других техник модель будет «лениться» предсказывать всё как норму.
3. **Метрика №1 для безопасности** — **Recall класса 0** (полнота по нарушениям); accuracy одна может обманывать при дисбалансе.
4. **Пайплайн честный** — импутация и scaler обучаются **только на train**, иначе завышенное качество на test.
5. На объединении **трёх доменов** сильный recall по нарушениям обычно у **Gradient Boosting** (см. снимок выше); **Random Forest** близок по качеству; **логистическая регрессия** — простой baseline.
