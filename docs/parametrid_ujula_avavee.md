# Показатели воды: открытая вода (supluskoha) и бассейны / ujula / СПА (basseinid)

В открытых водоёмах и в закрытых бассейнах контролируют **разные наборы** показателей: на пляже важны прежде всего **фекальные индикаторы** и общее состояние водоёма; в ujula и СПА — **дезинфекция (хлор)**, **мутность**, **оппортунистические бактерии** и признаки органической нагрузки.

Источник данных в проекте: XML Terviseamet (opendata), парсинг в `src/data_loader.py`. Краткие пороги: [`normy.md`](normy.md). Соответствие RU/ET/EN: [`glosarij.md`](glosarij.md).

**Питьевая вода (veevark)** здесь не разбирается: это отдельный регламент и другой набор параметров (см. `normy.md` и `_veevark_naitaja_col` в коде).

---

## 1. Открытые места купания — домен `supluskoha`

Море, озёра, реки и другие **supluskohad**. Нормативная логика ориентирована на EU Bathing Water Directive (классы качества по E. coli и энтерококкам; подробности в `normy.md`).

| Колонка в коде | Типичное имя в XML (`nimetus`) | Единица (часто) | Зачем измеряют |
|----------------|--------------------------------|-----------------|----------------|
| `e_coli` | Escherichia coli | КОЕ/100 мл | Индикатор **свежего фекального** загрязнения. Рост значений — риск кишечных инфекций после купания. |
| `enterococci` | Enterokokid / soole enterokokid / intestinal enterococci | КОЕ/100 мл | Индикатор фекального загрязнения, **устойчивее** в воде, чем E. coli; в директиве по купанию используется вместе с E. coli для классификации. |
| `ph` | pH | безразм. | Кислотность воды. Сильные отклонения могут раздражать кожу и глаза; для открытой воды чаще отражает геохимию водоёма, а не «безопасность как в бассейне». |
| `transparency` | Läbipaistvus | м (как в данных) | **Прозрачность** (видимая глубина / диск Secchi и т.п., в зависимости от методики в протоколе). Характеризует взвесь, водоросли, общее «состояние» воды для купания; не путать с **мутностью в NTU**, которая используется в veevärk и basseinid. |

**Особенность парсера:** показатель **coli-laadsed** (колиформы общие) в XML для supluskoha **намеренно не маппится** в колонку (`_supluskoha_naitaja_col` возвращает `None`): для купания в фокусе E. coli и энтерококки по директиве; общие колиформы в этом контексте не выводятся в датасет проекта.

В XML могут быть **другие** `naitaja`, не перечисленные выше; они не попадают в текущие колонки `DataFrame`.

## 2. Бассейны, ujula, СПА — домен `basseinid`

Закрытые или управляемые объекты: **bassein**, **ujula** (плавательный бассейн), зоны СПА. Вода **рециркулирует**, дезинфицируется (обычно хлор), нагрузка от людей высокая — поэтому набор показателей **шире по химии и микробиологии бассейна**, чем у открытого водоёма.

Нормативная сводка для части показателей: таблица «Basseinid» в [`normy.md`](normy.md). Ниже — смысл каждого поля **в коде проекта**.

### 2.1. Микробиология

| Колонка | XML / смысл | Единица | Описание |
|---------|---------------|---------|----------|
| `e_coli` | Escherichia coli | КОЕ/100 мл | Как и на открытой воде — маркер фекального сброса; в бассейне **не должно быть**; в нормативной таблице для бассейнов — **0**. |
| `coliforms` | coli-laadsed / колиформы | КОЕ/100 мл | Общие колиформные бактерии. Показывают гигиеническое качество воды и работу фильтрации/дезинфекции. |
| `enterococci` | enterokokid | КОЕ/100 мл | Фекальный индикатор; в закрытой системе указывает на сбои очистки или загрязнение. |
| `pseudomonas` | Pseudomonas (часто *P. aeruginosa*) | КОЕ/100 мл | **Оппортунистический патоген**, типичен для влажных сред и биоплёнок в трубопроводах; для бассейнов и СПА — важный показатель безопасности. |
| `staphylococci` | stafülokokid / Staphylococcus | КОЕ/100 мл | Связаны с **микрофлорой кожи** посетителей; контролируется верхним пределом (см. `normy.md`). |
| `colonies_37c` | kolooniate arv … 37 °C (формулировка в XML) | по методике | **Общее число колоний** при инкубации при температуре тела. Отражает общую гетеротрофную нагрузку, не специфичен к одному виду. |

### 2.2. Дезинфекция и химия бассейна

| Колонка | XML / смысл | Единица | Описание |
|---------|---------------|---------|----------|
| `free_chlorine` | vaba kloor | мг/л | **Свободный активный хлор** — основной дезинфицирующий агент. Слишком мало: риск микробов; слишком много: раздражение кожи и глаз. Допустимый **интервал** см. `normy.md`. |
| `combined_chlorine` | seotud kloor | мг/л | **Связанный хлор** (хлорамины и др.). Слабее по дезинфекции, даёт «хлорный запах», раздражение; нормируется **верхней границей**. |
| `ph` | pH | безразм. | В бассейнах узкий интервал (см. `normy.md`): влияет на эффективность хлора и комфорт. |
| `ammonium` | ammoonium | мг/л | Аммоний-азот. В бассейне связан с **мочой, потом**, реакцией с хлором и образованием хлораминов (рост `combined_chlorine`). |
| `nitrates` | nitraatioon | по единице в XML | В opendata для бассейнов это **не** то же самое, что «нитраты питьевой воды», а показатель, связанный с **нитратированием / балансом азота** в системе (название в протоколе — nitraatioon). Интерпретация — строго по методике лаборатории и регламенту. |

### 2.3. Прозрачность, цвет, органика

| Колонка | XML / смысл | Единица | Описание |
|---------|---------------|---------|----------|
| `turbidity` | hägusus | NTU | **Мутность** — взвешенные частицы. В бассейне низкая норма (см. `normy.md`): грязная вода хуже фильтруется и хуже обеззараживается. |
| `color` | värvus | часто мг Pt/л | **Цветность** воды; органика, техногенные вещества, продукты окисления. |
| `oxidizability` | oksüdeeritavus | по методике | **Окисляемость** (органическое вещество, потребление окислителя). Косвенно отражает органическую нагрузку и «старение» воды в контуре. |

### 2.4. Есть в нормативе Terviseamet, но не в текущем парсере opendata

| Параметр | Примечание |
|-----------|------------|
| **Температура воды** | В `normy.md` для basseinid указано **≤ 30 °C**; в `data_loader.py` для opendata **нет** извлечения температуры — если появится в XML как отдельный `naitaja`, его можно добавить в `_basseinid_naitaja_col`. |

## 3. Сводка: чего нет «у соседа»

| Показатель | Открытая вода (`supluskoha`) | Бассейн / ujula (`basseinid`) |
|------------|------------------------------|-------------------------------|
| `transparency` (läbipaistvus, м) | да | нет в парсере |
| `turbidity` / `color` (NTU, Pt) | нет | да |
| `free_chlorine` / `combined_chlorine` | нет | да |
| `pseudomonas`, `staphylococci`, `colonies_37c` | нет | да |
| `oxidizability`, `nitrates` (nitraatioon) | нет | да |
| `e_coli` + `enterococci` как главные индикаторы купания | да | да (E. coli по норме 0; энтерококки контролируются) |
| `coliforms` | не маппится | да |

## 4. Связь с признаками ML

В `src/features.py` определены два словаря нормативов:

- **`NORMS`** — общие пороги: `e_coli`, `enterococci`, `turbidity` (4.0 NTU для veevark/joogivesi), `ph` (6.0–9.0) и химические параметры питьевой воды.
- **`NORMS_POOL`** — специфичные для бассейнов/ujula/СПА: `turbidity` (0.5 NTU), `staphylococci` (≤ 20 КОЕ/100 мл), `pseudomonas` (0), `free_chlorine` (0.5–1.5 мг/л), `combined_chlorine` (≤ 0.5 мг/л), `ph` (6.5–8.5). Источник: Sotsiaalministri 31.07.2019 määrus nr 49, Lisa 4. См. `docs/phase_10_findings.md` § R1 для эмпирической верификации.

Функция `add_ratio_features` автоматически выбирает нужный набор норм по значению столбца `domain`:

| Признак | Расчёт для `basseinid` | Расчёт для остальных |
|---|---|---|
| `turbidity_ratio` | `turbidity / 0.5` | `turbidity / 4.0` |
| `staphylococci_ratio` | `staphylococci / 20` | NaN |
| `pseudomonas_detected` | `1` если > 0 | NaN |
| `free_chlorine_deviation` | расстояние от [0.5, 1.5] | NaN |
| `combined_chlorine_ratio` | `combined_chlorine / 0.5` | NaN |
| `ph_deviation` | расстояние от [6.5, 8.5] | расстояние от [6.0, 9.0] |

Это исправляет системную проблему ложных тревог для бассейнов, при которой предыдущая версия кода использовала норму 4.0 NTU для мутности бассейнов (реальная норма 0.5 NTU) и не кодировала хлор и стафилококки вообще.
