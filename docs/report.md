# Отчёт: Прогнозирование нарушений норм качества воды в Эстонии

**Курс:** TalTech Masinõpe (Machine Learning)  
**Данные:** Terviseamet (Департамент здоровья Эстонии) — [vtiav.sm.ee](https://vtiav.sm.ee)  
**Период:** 2021–2026 · **Домены:** supluskoha (купальные места), veevark (водопровод)

---

## 1. Введение

Эстонский Департамент здоровья регулярно публикует результаты лабораторных проб воды из купальных мест и водопроводных сетей. Каждая проба содержит измерения микробиологических и химических параметров и оценку «vastab nõuetele» / «ei vasta nõuetele» (соответствует / не соответствует нормам).

**Задача:** построить бинарную классификационную модель, которая по измеренным параметрам предсказывает, нарушает ли проба воды нормативы (`compliant = 0` — нарушение, `compliant = 1` — норма).

**Практическая значимость:** автоматическая ранняя диагностика потенциально опасных проб до завершения полного лабораторного цикла; приоритизация проверок.

**Приоритетная метрика — Recall класса 0 (нарушений).**  
False Negative (FN) = модель предсказала «норма», а вода содержит E. coli или превышение по химии. Это наихудший исход: люди купаются в опасной воде или пьют загрязнённую воду.

## 4. Методология

### 4.1 Инженерия признаков

Из сырых 15 параметров формируются **53 признака** трёх типов:

1. **Исходные значения** (15 параметров): `e_coli`, `ph`, `iron`, ...
2. **Отношение к норме** (14 признаков): `e_coli_ratio = e_coli / 500`, `manganese_ratio = manganese / 0.05` и т.д. Значение > 1 означает нарушение.
3. **Индикаторы пропуска** (15 признаков): `iron_missing`, `ph_missing`, ... — бинарный флаг. Факт отсутствия измерения несёт информацию о типе пробы.
4. **Временные признаки**: `month`, `year`, `is_summer`, `season_*` (one-hot)
5. **Домен**: `domain_supluskoha`, `domain_veevark` (one-hot)

pH кодируется как `ph_deviation` — расстояние от допустимого диапазона [6.0–9.0]: `max(0, 6.0 - pH, pH - 9.0)`.

### 4.2 Обработка пропусков и масштабирование

1. Признаки без ни одного значения в train → 0
2. `SimpleImputer(strategy='median')` — fit только на train
3. `StandardScaler` — fit только на train, transform на test

### 4.3 Разбивка данных

```
train: 80% = 30 875 проб  |  test: 20% = 7 719 проб
stratify=True (сохраняем долю классов)
random_state=42
```

### 4.4 Модели

| Модель | Ключевые параметры |
|--------|-------------------|
| Logistic Regression | `class_weight='balanced'`, `max_iter=2000` |
| Random Forest | `n_estimators=200`, `max_depth=12`, `class_weight='balanced'` |
| **Gradient Boosting** | `n_estimators=200`, `max_depth=4`, `learning_rate=0.1`, `sample_weight` |

## 5. Результаты

### 5.1 Метрики

| Модель | Accuracy | Precision (нарушение) | **Recall (нарушение)** | F1 (нарушение) | ROC-AUC |
|--------|----------|----------------------|----------------------|---------------|---------|
| Logistic Regression | 0.917 | 0.454 | 0.827 | 0.586 | 0.936 |
| Random Forest | 0.992 | 0.964 | 0.925 | 0.944 | 0.992 |
| **Gradient Boosting** | **0.993** | **0.946** | **0.954** | **0.950** | **0.994** |

**Лучшая модель: Gradient Boosting** — наивысший Recall на нарушениях (0.954) при точности 0.946 и ROC-AUC 0.994.

### 5.2 Confusion Matrix (Gradient Boosting, test set = 7 719 проб)

```
                  Pred: Норма   Pred: Нарушение
True: Нарушение      523           25  ← FP (лишняя тревога)
True: Норма           30         7141  ← FN (пропущены опасные пробы!)
```

- **FN = 30** пропущенных нарушений из 548 (5.5%) — допустимо для данной задачи
- **FP = 25** ложных тревог — незначительная нагрузка на лабораторию

Для сравнения: Logistic Regression пропускает **95 нарушений** (17.3%).

### 5.3 Наивный baseline

Модель "всегда говорить норма": Accuracy = 92.9%, но Recall(0) = **0%** — не обнаруживает ни одного нарушения. Наша модель GB превосходит baseline по всем значимым метрикам.

| `enterococci_ratio` | 0.055 | Отношение к норме (200 КОЕ/100 мл) |
| `manganese` | 0.053 | Концентрация марганца |
| `iron` | 0.053 | Концентрация железа |
| `ph_missing` | 0.048 | Факт отсутствия измерения pH |
| `manganese_ratio` | 0.047 | Отношение марганца к норме (0.05 мг/л) |

**Ключевые инсайты:**
- **Coliforms** (77%) — самый важный признак. Наличие колиформных бактерий — прямой индикатор фекального загрязнения.
- **Индикаторы пропуска** (`color_missing`, `ph_missing`) входят в топ-10: факт того, что параметр не измерялся, зависит от типа пробы и коррелирует с доменом/причиной пробы.
- **Ratio-признаки** (iron_ratio, manganese_ratio) более информативны, чем сырые значения — нормализация к норме устраняет масштабный эффект.
- **e_coli** только на 14-м месте: низкая средняя концентрация (медиана = 0), но когда превышает — сильное нарушение.

## 7. Ограничения

1. **Отсутствие географии.** Поле `maakond` (уезд) отсутствует в opendata XML — нет возможности строить географические признаки.

2. **Высокий процент пропусков.** Нитраты, нитриты, хлориды, сульфаты измеряются лишь в 5–7% проб veevark. Медианная импутация — грубое приближение; лучше использовать модели, поддерживающие пропуски нативно (XGBoost, LightGBM).

3. **Утечка данных в county_encoded.** LabelEncoder для `county` обучен на всём датасете до split (отсутствует в данных, поэтому некритично, но является методологическим нарушением).

4. **Один источник данных.** Только два домена из пяти (`supluskoha`, `veevark`). Данные бассейнов (`basseinid`), питьевых источников (`joogivesi`), минеральных вод (`mineraalvesi`) не используются.

5. **Нет темпоральной валидации.** Train/test разбивка случайная, а не по времени. Правильнее: train на 2021–2023, test на 2024–2025.

6. **Нет калибровки вероятностей.** Для систем раннего предупреждения важны откалиброванные вероятности (`CalibratedClassifierCV`).

## 8. Выводы и следующие шаги

### Выводы

- Задача решаема с высокой точностью: **Gradient Boosting достигает Recall(0) = 0.954** при ROC-AUC = 0.994.
- Главные предикторы нарушений: **coliform-бактерии**, **pH**, **железо**, **марганец** и **энтерококки**.
- Индикаторы пропуска (`_missing`) — неожиданно ценные признаки: они кодируют тип и цель пробы.
- Ratio-признаки (отношение к норме) улучшают интерпретируемость и производительность модели.
- Лето значимо хуже весны по качеству купальных вод (8.5% vs 1.8% нарушений).
